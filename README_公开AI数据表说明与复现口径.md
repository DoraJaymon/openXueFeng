# 公务员专业选择公开 AI 数据表说明与复现口径

本文档说明 `ai_public_tables` 目录下四张公开数据表、配套可视化图表、关键指标和数据处理口径。目标是让普通用户可以把这些聚合数据交给 GPT、豆包、Qwen 等 AI 工具，用来回答“几个专业哪个更适合考公务员”“哪些专业类机会更多”“省考中哪些专业更占优”等问题。

公开表只保留聚合后的招录人数指标，不提供岗位明细、原始职位表、行级解析结果和人工复核细节。

## 1. 先理解职位表里的现实问题

公务员职位表里的专业要求不是统一格式，不能简单按专业名称搜索。常见情况包括：

| 职位表写法 | 示例 | 处理难点 |
|---|---|---|
| 直接点名专业 | `会计学`、`030101K法学` | 可以映射到单个标准本科专业。 |
| 点名专业类 | `法学类`、`计算机类`、`0301法学类` | 类内所有专业理论上共享这批机会。 |
| 点名学科门类 | `管理学门类`、`法学学科门类` | 范围很宽，不能和直接对口混为一谈。 |
| 多个专业或专业类并列 | `财政学类、金融学类、财会审计类` | 需要拆分后合并成可报集合。 |
| 专业不限 | `不限`、`专业不限`、`无专业限制` | 对所有专业都一样，适合作为兜底机会单独看。 |
| 相关/相近/近似专业 | `英语相关专业`、`会计学、审计学、财务管理相关专业` | 需要人工确认扩展范围，不能无限扩大。 |
| 非标准专业类 | `财会审计类`、`法律类`、`财务财会类` | 不是教育部本科专业目录里的标准名称，需要人工映射。 |
| 学历分段 | `本科：会计学；研究生：会计` | 本项目只取本科可报部分。 |
| 附带资格或排除条件 | `法学类（需取得法律职业资格证书A证）`、`公安学类（不含...）` | 专业集合和资格条件要分开处理。 |

因此，本项目不是只做“字符串匹配”，而是先把职位专业要求映射到标准本科专业、专业类和学科门类，再做聚合统计。

## 4. 字段说明

### 4.1 专业表

适用文件：

- `01_全部考试_专业招录指标.csv`
- `03_省考_专业招录指标.csv`

| 字段 | 含义 |
|---|---|
| `专业` | 本科专业名称，例如“会计学”“法学”。 |
| `专业代码` | 本科专业代码，例如 `120203K`。 |
| `所属专业类` | 该专业所属专业类，例如“工商管理类”。 |
| `所属学科门类` | 该专业所属学科门类，例如“管理学”。 |
| `专业相关招录人数` | 排除专业不限后的专业相关机会总量。它包括直接对口、专业类共享、学科门类共享、非标准专业映射、相关/相近扩展等内部来源。 |
| `直接对口招录人数` | 职位要求明确点到该专业名称或专业代码带来的招录人数。 |
| `专业类共享招录人数` | 职位要求明确点到该专业所属专业类时，该专业作为类内成员共享到的招录人数。 |
| `学科门类共享招录人数` | 职位要求明确点到该专业所属学科门类时，该专业共享到的招录人数。 |
| `专业不限招录人数` | 专业要求为“不限”“专业不限”“无专业限制”等时带来的招录人数。 |

使用建议：

- 比较几个专业时，先看 `专业相关招录人数`，再看 `直接对口招录人数`。
- 如果某专业 `专业相关招录人数` 高但 `直接对口招录人数` 低，说明它更多依赖专业类、学科门类、非标准映射或相关/相近扩展。
- `专业不限招录人数` 是所有专业共享的兜底机会，不能证明某个专业本身更对口。

字段归类例子，假设每个示例职位招录 2 人：

- 职位要求写 `会计学`：计入会计学的 `直接对口招录人数` +2，同时计入会计学的 `专业相关招录人数` +2。
- 职位要求写 `120203K会计学、120204财务管理`：计入会计学和财务管理的 `直接对口招录人数` 各 +2，也计入这两个专业的 `专业相关招录人数` 各 +2。
- 职位要求写 `工商管理类`：工商管理类下所有本科专业都会计入 `专业类共享招录人数` +2，也计入这些专业的 `专业相关招录人数` +2；但不计入这些专业的 `直接对口招录人数`。
- 职位要求写 `管理学门类` 或 `管理学学科门类`：管理学门类下所有本科专业都会计入 `学科门类共享招录人数` +2，也计入 `专业相关招录人数` +2；但不计入 `直接对口招录人数` 或 `专业类共享招录人数`。
- 职位要求写 `财会审计类`：这是非标准专业簇，会通过人工映射进入会计学、财务管理、审计学等相关专业的 `专业相关招录人数`；但通常不会被当作“直接点名某个专业”。
- 职位要求写 `会计学、审计学、财务管理相关专业`：列出的核心专业会进入 `专业相关招录人数`，人工确认的相关/相近扩展专业也会进入 `专业相关招录人数`；其中被明确列出的专业才可能进入 `直接对口招录人数`。
- 职位要求写 `专业不限` 或 `不限`：只计入所有专业的 `专业不限招录人数`，不计入 `专业相关招录人数`。

读表数值例子，以下数值来自 `01_全部考试_专业招录指标.csv`：

- `会计学`：`专业相关招录人数` 为 18832，`直接对口招录人数` 为 1919，`专业类共享招录人数` 为 7042。读法是：会计学总体机会很高，既有明确点名机会，也大量受益于工商管理类、财会审计类等共享或映射口径。
- `法学`：`专业相关招录人数` 为 13311，`直接对口招录人数` 为 1587，`专业类共享招录人数` 为 10426。读法是：法学的很多机会来自“法学类”等专业类共享，不能只看职位是否直接写了“法学”两个字。
- `计算机科学与技术`：`专业相关招录人数` 为 11835，`直接对口招录人数` 为 1052，`专业类共享招录人数` 为 9547。读法是：该专业机会规模大，但相当一部分来自“计算机类”整体需求。
- `英语`：`专业相关招录人数` 为 1706，`直接对口招录人数` 为 1474，`专业类共享招录人数` 为 76。读法是：英语的总量不如会计、法学、计算机，但其中直接点名占比较高，说明它的机会更集中在明确要求英语的岗位上。
- 如果某专业的 `专业不限招录人数` 很高，不要据此判断它“专业优势强”。专业不限是所有专业共享的背景机会，真正比较专业差异时应优先看 `专业相关招录人数` 和 `直接对口招录人数`。

### 4.2 专业类表

适用文件：

- `02_全部考试_专业类招录指标.csv`
- `04_省考_专业类招录指标.csv`

| 字段 | 含义 |
|---|---|
| `专业类` | 本科专业类名称，例如“法学类”“计算机类”。 |
| `专业类代码` | 本科专业类代码，例如 `0301`。 |
| `所属学科门类` | 该专业类所属学科门类。 |
| `包含专业数量` | 该专业类下包含的本科专业数量。 |
| `本类总可报招录人数` | 类内任意专业可报到的招录人数并集。同一职位即使覆盖类内多个专业，也只计一次。 |
| `直接面向本类招录人数` | 职位要求直接写到该专业类名称或代码时带来的招录人数。 |
| `机会集中程度` | 描述该专业类内部机会是否集中在少数专业上。 |

使用建议：

- 比较专业类时，先看 `本类总可报招录人数`，再看 `直接面向本类招录人数`。
- `本类总可报招录人数` 高，说明这个类整体能碰到的机会多。
- `直接面向本类招录人数` 高，说明招录单位经常直接把这个专业类作为招聘对象。
- `机会集中程度` 高时，要回到专业表看类内具体专业，不能只看专业类总量。

字段归类例子，假设每个示例职位招录 2 人：

- 职位要求写 `法学类`：计入法学类的 `直接面向本类招录人数` +2，也计入法学类的 `本类总可报招录人数` +2。
- 职位要求写 `法学、知识产权`：这两个专业都属于法学类，因此法学类的 `本类总可报招录人数` +2；但职位没有直接写 `法学类`，所以不计入 `直接面向本类招录人数`。
- 职位要求写 `法学类、公安学类`：法学类和公安学类的 `本类总可报招录人数` 各 +2，两个专业类的 `直接面向本类招录人数` 也各 +2。
- 职位要求写 `会计学、财务管理、审计学`：这三个专业都属于工商管理类，所以工商管理类的 `本类总可报招录人数` +2；但同一个职位只计一次，不会因为覆盖 3 个类内专业就加 6。
- 职位要求写 `财会审计类`：如果人工映射到工商管理类下若干财会审计核心专业，则工商管理类会获得 `本类总可报招录人数`；但是否计入 `直接面向本类招录人数`，取决于这个非标准表达是否被映射为标准专业类还是具体专业簇。
- 职位要求写 `管理学门类`：管理学下多个专业类都会获得 `本类总可报招录人数`；但它不是直接写某个专业类，因此不计入某个专业类的 `直接面向本类招录人数`。
- 职位要求写 `专业不限`：不用于判断某个专业类的专业对口优势，不应和 `直接面向本类招录人数` 混在一起解释。

读表数值例子，以下数值来自 `02_全部考试_专业类招录指标.csv`：

- `工商管理类`：`本类总可报招录人数` 为 25337，`直接面向本类招录人数` 为 7707，`机会集中程度` 为“较集中”。读法是：工商管理类总体机会很大，但类内机会并不完全平均，选择具体专业时需要回到专业表看会计学、财务管理、审计学等专业的差异。
- `法学类`：`本类总可报招录人数` 为 18846，`直接面向本类招录人数` 为 12535，`机会集中程度` 为“较均衡”。读法是：法学类不仅整体机会多，而且被直接作为专业类要求的招录人数也高，类内机会相对不那么集中。
- `计算机类`：`本类总可报招录人数` 为 18875，`直接面向本类招录人数` 为 10252，`机会集中程度` 为“较均衡”。读法是：计算机类整体和直接面向本类的机会都较多，类内多个专业都能共享较多专业类机会。
- `外国语言文学类`：`本类总可报招录人数` 为 7271，`直接面向本类招录人数` 为 243，`机会集中程度` 为“高度集中”。读法是：这个专业类看起来有一定总量，但直接面向整个专业类的机会少，且机会可能集中在英语等少数专业上。
- `包含专业数量` 也要结合集中程度看。一个专业类包含专业很多时，类内平均到每个专业的机会未必多；如果同时 `机会集中程度` 高，普通用户更应该查自己的具体专业，而不是只看专业类名称。

## 6. 关键指标和新概念

### 6.1 本类总可报招录人数

这是专业类表的主指标。它不是把类内所有专业的机会相加，而是做并集统计。

例子：某职位要求“会计学、财务管理、审计学”，招录 2 人，这三个专业都属于工商管理类。对工商管理类来说，这个职位在 `本类总可报招录人数` 中只增加 2 人，而不是增加 6 人。

这样可以避免一个职位覆盖同类多个专业时被重复放大。

### 6.2 直接面向本类招录人数

表示职位要求直接写到专业类名称或代码。

例子：

- 写“法学类”，计入法学类的 `直接面向本类招录人数`。
- 写“法学、知识产权”，虽然两个专业属于法学类，但不计入 `直接面向本类招录人数`，只计入法学类的总可报机会。

这个指标用来区分“招录单位明确要这个专业类”和“类内某些专业碰巧能报”。

### 6.3 机会集中程度

机会集中程度描述专业类内部机会是否集中在少数专业上。

计算方式：

`集中度数值 = 类内最高专业相关招录人数 / 类内专业相关招录人数均值`

公开表不暴露具体数值，只给等级：

| 集中度数值 | 公开等级 |
|---:|---|
| 小于或等于 0 | 无数据 |
| 小于 1.25 | 较均衡 |
| 1.25 到 1.75 | 略集中 |
| 1.75 到 2.5 | 较集中 |
| 2.5 及以上 | 高度集中 |

解释：

- `较均衡`：类内多数专业机会差距不大。
- `较集中` 或 `高度集中`：该专业类看起来机会多，但可能主要集中在少数专业上。

### 6.4 专业类独占指数

独占指数用于可视化图 4，不在四张公开表中直接提供。

计算思路：

1. 对每个职位，先看它覆盖了几个专业类。
2. 如果一个职位只覆盖 1 个专业类，则招录人数全部分给这个专业类。
3. 如果一个职位覆盖 4 个专业类，则招录人数按 1/4 分给每个专业类。
4. 汇总后得到该专业类的“折算独占招录人数”。
5. 用折算独占招录人数除以该专业类的总可报招录人数。

公式：

`专业类独占指数 = Σ(职位招录人数 / 该职位覆盖的专业类数量) / 本类总可报招录人数`

解释：

- 独占指数越高，说明机会越专属于这个专业类。
- 独占指数越低，说明该专业类的机会经常和很多其他专业类共同分享。

例子：一个职位招 8 人，只覆盖法学类，则法学类折算得到 8 人；另一个职位招 8 人，同时覆盖法学类、财政学类、计算机类、工商管理类，则每个专业类只折算得到 2 人。

## 7. 映射和统计处理

### 7.1 标准专业映射

职位表中的专业要求会被映射到标准本科专业目录中的：

- 专业。
- 专业类。

匹配时优先使用专业代码和标准名称，再处理专业类、学科门类、并列项、括号项、学历分段和特殊表达。

### 7.2 非标准专业映射

非标准专业是职位表中出现的、不是标准本科专业目录名称的表达。

人工映射表：

`专业分析/公务员专业分析/manual_mappings/国考江苏非标准专业类映射_含窄口径.csv`

例子：

- `法律类` 映射到标准 `法学类`。
- `财务财会类` 映射到会计学、财务管理、财务会计教育、内部审计等核心专业。
- `财会审计类` 映射到会计学、财务管理、审计学、财务会计教育、内部审计、工程审计等核心专业簇。

原则：

- 能映射到标准专业类的，映射到标准专业类。
- 更像专业簇的，映射到人工确认的核心专业集合。
- 不因为名称相似就自动扩展到整个大类。
- 只使用 `manual_status = proposed` 的人工确认映射。

### 7.3 相关/相近/近似专业映射

职位表中出现 `相关专业`、`相近专业`、`近似专业` 时，不能直接扩展到整个专业类。

人工扩展表：

`专业分析/公务员专业分析/manual_mappings/本科相关近似基础项去重清单.csv`

只有当该表中 `manual_status = proposed` 且有 `manual_related_or_approx_codes` 时，才会进行相关/相近扩展。

例子：

- `英语相关专业` 扩展到英语、翻译、商务英语，不扩展到整个外国语言文学类。
- `会计学、审计学、财务管理相关专业` 扩展到财会审计相关专业簇。
- `法学相关专业` 扩展到人工确认的法学相关专业簇，不默认扩展到全部法学类专业。

### 7.4 窄口径和宽口径在这里的作用

窄口径、宽口径是处理流程中的内部口径，不是普通用户最需要优先理解的指标。

- 窄口径：保留职位要求中明确、核心、可解释的专业范围，避免“相关/相近/近似”无限扩大。
- 宽口径：在窄口径基础上，对人工确认的相关/相近/近似专业做有限扩展。

四张公开 AI 表使用宽口径生成专业和专业类的主统计。图 1 也使用真宽口径来源分解，但为了强调“真正对口”和“共享机会”的区别，图 1 不纳入 `专业不限` 和 `学科门类共享`，橙色部分只保留专业类、相关/相近扩展和非标准专业映射带来的共享机会。

## 9. 用户可以用这些数据问 AI 什么问题

把四张 CSV 和本文档交给 AI 后，适合让 AI 做“专业选择辅助判断”。下面是可以直接照抄的提问方式。

### 9.1 比较几个具体专业

适合问题：

- `我在会计学、法学、计算机科学与技术三个专业中选，哪个更适合以后考公务员？请用专业相关招录人数、直接对口招录人数、专业类共享招录人数分别比较。`
- `我想在英语和汉语言文学之间选一个更适合考公务员的专业，请看全部考试表和省考表分别给建议。`
- `这几个专业里，哪些是总机会多但直接对口弱？哪些是直接对口更强？`
- `请把我的备选专业按公务员招录机会分成推荐、一般、谨慎三档，并解释依据。`

AI 应该怎么回答：

- 先说明使用的是全部考试表还是省考表。
- 先比较 `专业相关招录人数`。
- 再比较 `直接对口招录人数`。
- 最后解释该专业是否依赖 `专业类共享招录人数`、`学科门类共享招录人数` 或 `专业不限招录人数`。

### 9.2 比较专业类

适合问题：

- `法学类、计算机类、财政学类、工商管理类，哪个专业类更适合考公务员？`
- `哪些专业类的本类总可报招录人数高，同时直接面向本类招录人数也高？`
- `哪些专业类看起来机会多，但机会集中程度高，选具体专业时要谨慎？`
- `请找出省考表中更值得关注的前 10 个专业类，并说明它们是靠总量、直接面向本类，还是类内集中度取胜。`

AI 应该怎么回答：

- 先看 `本类总可报招录人数`，判断专业类整体机会。
- 再看 `直接面向本类招录人数`，判断招录单位是否经常直接写这个专业类。
- 再看 `机会集中程度`，提醒用户是否需要回到专业表检查具体专业。

### 9.3 专门问省考

适合问题：

- `如果我主要考虑省考，不考虑国考，法学、会计学、计算机科学与技术哪个更好？`
- `省考里哪些专业类更有优势？请只使用省考专业类表。`
- `某个专业在全部考试里强，但在省考里是否仍然强？请对比全部考试表和省考表。`

AI 应该怎么回答：

- 用户明确问省考时，只用 `03_省考_专业招录指标.csv` 和 `04_省考_专业类招录指标.csv`。
- 如果要对比国考 + 省考总样本，则再引用 `01` 和 `02` 表。
- 不能把全部考试表里的结论直接当成省考结论。

### 9.4 找风险和短板

适合问题：

- `我的备选专业中，哪些公务员招录机会最弱？`
- `哪些专业主要靠专业不限，专业本身对口性不强？`
- `哪些专业类机会集中程度高，说明类内专业差异很大？`
- `请指出这几个专业中最不建议仅为了考公务员而选择的专业，并说明是总量低、直接对口低，还是过度依赖共享机会。`

AI 应该怎么回答：

- 不只给排名，还要解释风险来源。
- 区分“总量低”和“直接对口低”。
- 对机会集中程度高的专业类，提醒用户继续查具体专业。

### 9.5 做选专业决策辅助

适合问题：

- `我更看重省考，且不想选择太偏的专业。请根据省考表给我推荐几个专业方向。`
- `如果我在专业兴趣差不多的情况下，希望公务员选择面更广，应该优先考虑哪些专业或专业类？`
- `请基于这些表，给高中生解释为什么不能只看“专业不限”来判断一个专业好不好考公。`
- `请把这些数据转成给家长看的解释：为什么法学类、财政学类、计算机类、工商管理类经常被认为考公机会较多？`

AI 应该怎么回答：

- 把数据解释成方向性证据，而不是录取承诺。
- 同时提醒专业选择还要考虑兴趣、课程难度、学校层次、就业面、深造路径等非公务员因素。
- 不要说“某专业一定更容易上岸”，只能说“从招录机会口径看更有优势”。

### 9.6 不适合问的问题

不适合问题：

- `我能不能报某个具体岗位？`
- `这个专业上岸概率是多少？`
- `今年某岗位进面分是多少？`
- `哪个专业竞争比最低？`
- `某个单位资格审查一定会不会通过？`

原因是公开表只有聚合招录人数，不包含具体岗位条件、报名人数、进面分、资格审查结果等信息。

## 10. 给 AI 工具使用时的建议提示词

可以把四张 CSV 和本文档一起给 AI，并要求：

```text
你是公务员专业选择助手。请只根据我提供的公开数据表和字段说明回答。
比较专业时，优先看“专业相关招录人数”，再看“直接对口招录人数”，并说明是否依赖专业类共享、学科门类共享或专业不限。
比较专业类时，优先看“本类总可报招录人数”，再看“直接面向本类招录人数”和“机会集中程度”。
如果用户问省考，请使用省考表；如果没有指定考试类型，可以先用全部考试表，并提醒省考口径可能不同。
不要把招录人数说成录取概率、上岸概率或竞争难度。
```

## 11. 限制

- 公开表是聚合数据，不支持反查具体岗位。
- 招录人数不是报考人数、竞争比、进面分或上岸概率。
- 专业不限机会对所有专业都一样，不能证明某个专业本身更对口。
- 相关/相近/近似扩展依赖人工映射，适合方向性分析，不等同于最终资格审查结论。
- 非标准专业映射采用保守原则，但仍可能与个别地区、个别单位的实际审核口径不同。
- 省考表是云南、山东、湖北、江苏四个省考样本的合并口径，不代表全国所有省份。
